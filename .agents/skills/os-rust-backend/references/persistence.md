# Persistence

`*-persistence` owns Postgres. It hides the schema and SQL from the rest of the
code; everything outside this crate sees only domain types and `*Params` query
inputs. Stick to a small, predictable shape so the layer doesn't grow tendrils.

## What's in the crate

```
crates/persistence/
└── src/
    └── lib.rs    # connect(), run_migrations(), repository fns, row + params structs
```

Both repos keep `persistence` as a single `lib.rs` for now. As the schema grows,
split by aggregate (`users.rs`, `apps.rs`, `grants.rs`) — but always with
public functions, not impl-blocks-on-structs-with-private-state. The pool is
the only shared state.

## Connect + migrate

```rust
use library_config::DatabaseConfig;
use sqlx::{PgPool, postgres::PgPoolOptions};
use thiserror::Error;

pub async fn connect(config: &DatabaseConfig) -> Result<PgPool, PersistenceError> {
    PgPoolOptions::new()
        .max_connections(config.max_connections)
        .connect(&config.url)
        .await
        .map_err(PersistenceError::Connect)
}

pub async fn run_migrations(pool: &PgPool) -> Result<(), PersistenceError> {
    sqlx::migrate!("../../migrations")
        .run(pool)
        .await
        .map_err(PersistenceError::Migrate)
}

#[derive(Debug, Error)]
pub enum PersistenceError {
    #[error("connect Postgres: {0}")]
    Connect(#[source] sqlx::Error),
    #[error("run migrations: {0}")]
    Migrate(#[source] sqlx::migrate::MigrateError),
    #[error("query failed: {0}")]
    Query(#[from] sqlx::Error),
    #[error("conflict on {field}")]
    Conflict { field: &'static str },
    #[error("not found")]
    NotFound,
}
```

`sqlx::migrate!` runs at compile time to embed every `.sql` file from the
migrations directory into the binary. Migrations apply via the binary's
`migrate` subcommand, run by the runtime as a pre-deploy step before the
new release accepts traffic.

## Migrations: numbered, additive, never edited after merge

```
api/migrations/
├── 0001_users.sql
├── 0002_apps.sql
├── 0003_credit_ledger.sql
└── 0004_action_grants.sql
```

Format: `NNNN_description.sql`. Numbers are zero-padded for sort order. Once
merged, a migration **is immutable** — fix forward with a new migration, never
edit a deployed one (sqlx tracks checksums; editing a deployed migration breaks
the next deploy).

Each migration is one transaction. Avoid `CONCURRENTLY` operations that can't
run in a transaction unless you really need them — and if you do, comment why.

## ID conventions

```sql
-- prefixed IDs are TEXT, generated by the app (uuidv7 or nanoid)
user_id     TEXT PRIMARY KEY,        -- "usr_<uuid7>"
app_id      TEXT PRIMARY KEY,        -- "app_<nanoid>"
grant_id    TEXT PRIMARY KEY,        -- "agt_<uuid7>"

-- monotonic surrogates for log-style tables
id          BIGSERIAL PRIMARY KEY,   -- credit_ledger, webhook_events

-- timestamps are always TIMESTAMPTZ
created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
expires_at  TIMESTAMPTZ            -- nullable for "never expires"
```

Prefixed text IDs are the rule for anything that appears on the wire — they're
self-describing in logs (`grant_id=agt_abc123` is obvious; `id=42` is not), and
the prefix makes copy-paste errors visible (`usr_abc` pasted into a field
expecting `app_*` is immediately wrong).

Generate them in Rust, not Postgres:

```rust
use uuid::Uuid;
fn new_user_id() -> String { format!("usr_{}", Uuid::now_v7().simple()) }
```

## Enum columns: `CHECK` constraint + matching Rust enum

Every enum-shaped column gets a `CHECK` constraint. No open `TEXT`. The SQL
values, the JSON wire values, and the value the Rust enum serializes to must
be **identical** — lowercase for single words, snake_case for multi-word.

```sql
-- migrations/0004_action_grants.sql
CREATE TABLE action_grants (
    grant_id     TEXT PRIMARY KEY,
    user_id      TEXT NOT NULL REFERENCES users(user_id),
    app_id       TEXT NOT NULL REFERENCES apps(app_id),
    action       TEXT NOT NULL CHECK (action ~ '^[a-z][a-z0-9_-]+\.[a-z][a-z0-9_-]+$'),
    cap_credits  BIGINT NOT NULL CHECK (cap_credits > 0),
    status       TEXT NOT NULL DEFAULT 'open'
                 CHECK (status IN ('open', 'settled', 'expired', 'revoked')),
    -- ...
);
CREATE INDEX action_grants_open_idx
    ON action_grants(user_id, app_id)
    WHERE status = 'open';
```

The matching Rust enum:

```rust
#[derive(Debug, Clone, Copy, sqlx::Type, serde::Serialize, serde::Deserialize, utoipa::ToSchema)]
#[serde(rename_all = "snake_case")]
#[sqlx(rename_all = "snake_case")]
#[sqlx(type_name = "TEXT")]
pub enum GrantStatus {
    Open,
    Settled,
    Expired,
    Revoked,
}
```

When you add a value: bump the migration (new `CHECK` constraint), update the
Rust enum, update the wire types in the FE, update the enum-casing spec table.
All in the same PR.

## Repository functions

The crate exposes free functions, not impl blocks on a "repository struct". The
pool flows in as a parameter; the function does one query (or one transaction):

```rust
use sqlx::PgPool;
use library_domain::User;

pub struct FindOrCreateUserParams {
    pub external_id: String,
    pub email: Option<String>,
    pub display_name: Option<String>,
}

pub async fn find_or_create_user(
    pool: &PgPool,
    params: FindOrCreateUserParams,
) -> Result<User, PersistenceError> {
    let mut tx = pool.begin().await?;

    if let Some(user) = sqlx::query_as!(
        UserRow,
        r#"SELECT user_id, external_id, email, display_name, created_at FROM users WHERE external_id = $1"#,
        params.external_id
    ).fetch_optional(&mut *tx).await? {
        tx.commit().await?;
        return Ok(user.into());
    }

    let user_id = format!("usr_{}", uuid::Uuid::now_v7().simple());
    let row = sqlx::query_as!(
        UserRow,
        r#"INSERT INTO users (user_id, external_id, email, display_name)
           VALUES ($1, $2, $3, $4)
           RETURNING user_id, external_id, email, display_name, created_at"#,
        user_id, params.external_id, params.email, params.display_name,
    ).fetch_one(&mut *tx).await?;

    tx.commit().await?;
    Ok(row.into())
}
```

Three conventions baked in:

1. **`*Params` struct for 3+ inputs** — matches [spec/struct-params.md](../../../spec/struct-params.md).
2. **`query_as!` with a row struct.** The compile-time check catches schema
   drift; the row struct is `pub(crate)` and converts into the public domain
   type. Don't export rows.
3. **One transaction per logical operation.** Read-then-write goes in `begin()`
   → `commit()`, not two independent statements.

## Row → domain conversion

Keep the row type private and convert at the crate boundary:

```rust
pub(crate) struct UserRow {
    user_id: String,
    external_id: String,
    email: Option<String>,
    display_name: Option<String>,
    created_at: chrono::DateTime<chrono::Utc>,
}

impl From<UserRow> for library_domain::User {
    fn from(row: UserRow) -> Self {
        library_domain::User {
            id: library_domain::UserId(row.user_id),
            external_id: library_domain::UserExternalId(row.external_id),
            email: row.email,
            display_name: row.display_name,
            created_at: row.created_at,
        }
    }
}
```

This is what lets the API and services layers stay free of column names.

## Transactions and serialization

For operations that read-modify-write under contention (the `/authorize` path
in `os-accounts` is the canonical example), use Postgres advisory locks
keyed on the contended entity:

```rust
sqlx::query!("SELECT pg_advisory_xact_lock(hashtext($1))", user_id)
    .execute(&mut *tx).await?;

// now read, check invariant, insert — all serial per-user
let balance: i64 = sqlx::query_scalar!(
    "SELECT COALESCE(SUM(amount), 0)::BIGINT FROM credit_ledger WHERE user_id = $1",
    user_id
).fetch_one(&mut *tx).await?.unwrap_or(0);

// ... apply check, insert grant, commit
```

`pg_advisory_xact_lock` releases at transaction end — no leak risk. The
canonical primitive when you need "one writer per user at a time" without
locking entire tables. Don't combine with `FOR UPDATE` on the user row;
the advisory lock alone is enough.

## Idempotency keys

Mutation endpoints that can be retried (charging credits, accepting webhooks,
processing checkout completions) take an `idempotency_key` and dedupe on it:

```sql
CREATE TABLE charges (
    transaction_id    TEXT PRIMARY KEY,           -- "txn_<uuid7>"
    grant_id          TEXT NOT NULL REFERENCES action_grants(grant_id),
    credits_settled   BIGINT NOT NULL CHECK (credits_settled >= 0),
    idempotency_key   TEXT NOT NULL,
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    UNIQUE (grant_id, idempotency_key)
);
```

On insert, catch the unique-violation, look up the existing row, and return it
with `idempotent_replay: true`:

```rust
match sqlx::query_as!(ChargeRow, "INSERT INTO charges (...) VALUES (...) RETURNING ...", ...)
    .fetch_one(&mut *tx).await
{
    Ok(row) => Ok((row, false)),
    Err(sqlx::Error::Database(db)) if db.is_unique_violation() => {
        let row = sqlx::query_as!(ChargeRow,
            "SELECT ... FROM charges WHERE grant_id = $1 AND idempotency_key = $2",
            grant_id, idempotency_key)
            .fetch_one(&mut *tx).await?;
        Ok((row, true))
    }
    Err(e) => Err(e.into()),
}
```

## Money: integers, no floats

Credits and currency are `BIGINT`, never `NUMERIC`-as-float / `DOUBLE PRECISION`.
For the OS Accounts model: `$1 = 1000 credits`, so a credit balance is just
a signed `BIGINT` summing the ledger. For other money tables (Stripe amounts in
cents), keep `BIGINT` cents and convert at the wire.

When you need decimal math (tax rates, percentages), use `BIGDECIMAL` via
`sqlx`'s `bigdecimal` feature. Never `f64`.

## Common persistence mistakes

- **Mutating an old migration.** It breaks every environment that already
  applied the old version. Always add a new migration.
- **Exposing `sqlx::Error` to API handlers.** Wrap it in `PersistenceError`
  inside this crate; the API layer maps `PersistenceError::NotFound` →
  `ApiError::NotFound`, everything else → `ApiError::Persistence` (logged,
  rendered as 5000).
- **Returning row structs instead of domain types.** Rows are an implementation
  detail of this crate; convert with `From`/`Into` at the boundary.
- **Two queries that should be one transaction.** `read_balance` then
  `insert_charge` without `tx.begin()` is a race waiting to happen.
- **Filtering with `WHERE created_at > NOW() - INTERVAL '1 hour'` in a hot
  path without an index.** If the column is queried, index it; if the
  predicate is conditional ("only open grants"), use a partial index.
- **`SELECT *` in a `query_as!`.** Always enumerate columns — the compile-time
  check needs to know exactly what the row struct expects.
- **Building dynamic SQL by string-concatenation.** Use sqlx's typed builders
  or write multiple query branches. String SQL is how SQL injection happens.
